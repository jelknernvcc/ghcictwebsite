Projects will be evaluated using the Group Presentation Rubric. From our class discussion, I'd like to try an experiment with this presentation.

We all understand that programming skills and understanding are distributed unevenly among our class members. The goal of a group project is to learn together. In that spirit, each group should select the member who was most responsible for writing the code. That member will not make the oral presentation of the project to the class, so it will be up to the other members to understand and present the group's implementation.
