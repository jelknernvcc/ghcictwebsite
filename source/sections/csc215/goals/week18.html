I'll begin class today by sharing and discussing several small programs that illustrate the use of null terminated strings in C.

Strings in C are arrays of chars terminated by a null character ('\0'). That means a string of length 10 can hold only 9 characters, since one element is occupied by the terminating null.

01_str_vs_char.c

The string.h library provides several useful functions for copying and comparing strings, two of which, strlen and strcmp, are used here:

02_string_lib_experiments.c

Adaptations to Our Textbook Source

We will need to make a few adaptations to the source code examples presented in our textbook since we do not have conio.h and since the gets function is deprecated.

We will replace calls to gets with calls to fgets. fgets requires two additional arguments, num, set to the maximum length of the string being read, and stream, and pointer to the file stream from which to read the string. Since we want this last value to be standard input, we will pass stdin for this last paramemter.

For an example of this adaptation, the call to: 

gets(str);

in programming example 1 on page 119 of our text will be replaced with: 

fgets(str, 100, stdin);

We also need a replacement for the function getch. I've updated console_utils.h to add a getch function. We'll take a few minutes during class to confirm that you can compile and run the example programs on page 119 of our text using these adaptations.
